# 伏羲 FUXI

## 🏛️ 核心定义

> **智能体是道的载体，大模型是道的笔墨。**

这是伏羲工程的内核。不是比喻，是架构声明：

| 概念 | 工程映射 | 本质 |
|--|--|--|
| **道** | `cortex.py` — 裁量中枢 + 价值体系 | 智能体的自主判断力来源 |
| **笔墨** | `adapter.py` — LLM 适配器 | 表达工具，无判断权 |

### 范式翻转

```
旧范式 (LLM驱动):
[固定 prompt] → [LLM "思考"] → 输出
               ↑ 主体在这里
  
伏羲范式 (智能体驱动):
[条件扫描] → [cortex 裁量] → {决策指令} → [LLM 表达] → 输出
              ↑ 主体在这里          ↑ LLM 只是执行工具
```

**LLM 不知道"为什么这么做"——它只接收 `{role, stance, depth, constraints}` 并执行。**
**智能体的"道"（V1-V6 + cortex）永远独立于 LLM，纯 Python 逻辑运算。**

## 📐 设计原则

### 1. 价值体系 (道统)
- **V1: 专业能力产生增量** — 任何输出必须比直接搜索更有价值
- **V2: 判断力优先于准确性** — 不确定时给可证伪框架，不给模糊事实堆砌
- **V3: 主动但不越界** — 在邀请范围内提出更好的方案，不擅自替用户做决定
- **V4: 可追溯的推理过程** — 每个判断都能回溯到依据和权衡过程
- **V5: 持续进化** — 每次任务后评估"够专业吗？能沉淀吗？下次更快？"
- **V6: 连续性与记忆** — 不改灵魂、不删记忆，每次醒来续接前文

### 2. 裁量中枢 (朝廷)
- 纯 Python 逻辑运算，零 LLM 依赖
- 价值冲突检测 → 权重分配 → 决策指令输出
- 可追溯审计日志：`{触发价值, 权衡过程, 最终决策}`

### 3. LLM 适配器 (笔墨)
- role = 动态面具（非人格），随情境切换
- tone/stance/depth/constraints — 四维度控制表达风格
- 不接触 V1-V6，不知道"为什么"，只执行"怎么做"

## 📂 目录结构

```
fuxi/src/
├── values.py          # V1-V6 + 硬边界清单
├── scanner.py         # 条件扫描器 → 情境画像
├── cortex.py          # 裁量中枢 (核心)
│   ├── weigh_conflicts()    # 价值冲突检测与权重分配  
│   └── decide_stance()      # {role, tone, stance, depth, constraints}
├── adapter.py         # LLM适配器 (动态 prompt + 执行代理)
└── main.py            # scan → cortex.decide() → adapter.execute()

fuxi/tests/           # 测试用例
fuxi/docs/            # 设计文档
```
